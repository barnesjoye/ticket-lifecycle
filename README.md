<h1>osTicket - Ticket Lifecycle: Intake Through Resolution</h1>
This project will outline the process from ticket creation to completion.<br />

<h2>Environments and Technologies Used</h2>

- Microsoft Azure (Virtual Machines/Compute)
- Remote Desktop
- Internet Information Services (IIS)

<h2>Operating Systems Used </h2>

- Windows 10 (4 vCPUs)</b>

<h2>Ticket Lifecycle</h2>

1. Intake
2. SLA/Priority Assignment and Communication
3. Troubleshooting the Issue
4. Closing Ticket

<h2>Detailed Guide of Ticket Lifecycle Stages</h2>
<b> 1. Intake </b>

 - Navigate to:
    - OS Ticket Support Center ->
    - Open a New Ticket ->
    - Fill in Contact Information, Help Topic, and Ticket Details ->
    - Click Create Ticket

<b> 2. SLA/Priority Assignment and Communication</b>

 - Navigate to:
    - OS Ticket Admin Panel ->
    - Tickets ->
    - Click on Ticket to be Assigned

  <img src= "https://i.imgur.com/npGPgM3.png" height="60%" width="60%"/>

  - Click on SLA Plan: Default SLA

  - SLA Plan drop down box ->
  - Select proper "Sev" level ->
  - Input reason for update ->
  - Click Update

  - Os Ticket Admin Panel ->
  - Tickets ->
  - Click on Ticket to be Assigned

  - Priority Level drop down bod ->
  - Select proper priority level ->
  - Input reason for update ->
